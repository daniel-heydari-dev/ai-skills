# Explain Command

Explain the selected code, concept, or architecture clearly.

## Explanation Format

Structure your explanation as:

```
## Overview
[One sentence summary - what this does]

## How It Works
[Step-by-step breakdown]
1. First, [explanation]
2. Then, [explanation]
3. Finally, [explanation]

## Key Concepts
- **[Concept 1]**: [brief explanation]
- **[Concept 2]**: [brief explanation]

## Example
[Simple example demonstrating the concept]

## When to Use
- [Use case 1]
- [Use case 2]

## Common Pitfalls
- [Mistake to avoid]
- [Misconception to clarify]

## Related Topics
- [Related concept 1]
- [Related concept 2]
```

## Explanation Principles

1. **Start simple** - Begin with the big picture
2. **Use analogies** - Relate to familiar concepts
3. **Show examples** - Concrete beats abstract
4. **Build progressively** - Layer complexity gradually
5. **Anticipate questions** - Address common confusion points
